Bruk av GitHub for Samarbeid: Utfordringer og Læring

For å samarbeide brukte vi GitHub. Vi var helt ukjente med denne plattformen, og det krevde derfor en betydelig innsats for å forstå både dens funksjonalitet og bruksområde. Gjennom omfattende selvstudier kom vi til erkjennelsen av at GitHub er et essensielt verktøy for å kunne jobbe sammen på et prosjekt og dele koder.

GitHub Utvikling

I det første bildet, figur 1 commit history, kan man se hvordan vi slet med å forstå GitHub og måtte prøve oss frem ved å commite ulike ting. I det andre bildet, figur 2 commit history, ser man imidlertid en forbedring. Det tok en stund før vi endelig forstod hvordan vi skulle bruke plattformen korrekt. Det tredje bildet, figur 3 changes, viser hvordan vi endra på navnenet på classene slik at de ikke skulle kræsje, og for å ha oversikt over hvilken del koden gjelder. F.eks. på bilde het classen først "parentcontainer", men ble endra til "studie-parentcontainer" som indikerer til at denne containeren tilhører Studieoppbygging delen. På det siste bildet, figur 4 changes, ser man hvordan vi endret kommentarer fra norsk til engelsk for å holde det ryddig i koden.

Utfordringer vi støtet på

GitHub er en meget nyttig plattform for programmerere, men som nybegynnere møtte vi på en rekke utfordringer.

  • Vi opplevde det som skremmende å merge sammen branchene våre fordi vi var bekymret for at koden kunne bli ødelagt eller at koder skulle overlappe hverandre
  • Det var ikke lett å forstå om vi skulle jobbe i én branch eller hver vår og merge alle sammen tilslutt. Så vi starta først med å jobbe i hver våre brancher og merge dem inn til main, men fikk masse "conflicts" og det tok kjempe lang å gå igjennom. Senere fant vi ut at det var mye enklere at alle jobber på samme branch og commiter underveis, det vil gjøre at alle som er med i den branchen får opp endringene som har blitt gjort ved å fetche og pulle inne på GitHub Desktop.
  • Videre var det mange ukjente begreper for oss, som for eksempel merge, branch, commit og repository, som gjorde at vi ikke fullt ut forstod hva vi trykket på eller hva hver funksjon gjorde.
  • Vi så på YouTube-videoer og spurte læringsassistenter om hjelp, men syntes ofte at forklaringene gikk for fort. Den eneste måten vi klarte å løse utfordringene på, var gjennom utprøving og feiling, til vi til slutt oppnådde en viss forståelse.
