Using GitHub for Collaboration: Challenges and Learning

To collaborate effectively, we utilized GitHub. As complete novices to this platform, it took considerable effort to understand its purpose and how to use it. Through extensive self-study, we realized that GitHub is a crucial tool for working together on a project and sharing our code with each other.

In the first image (Figure 1, Commit History), one can observe our initial struggles with understanding GitHub, where we had to experiment extensively. In the second image (Figure 2, Commit History), however, we can see significant improvement as we eventually grasped the correct usage of the platform. The third image (Figure 3, Changes) showcases the modifications we started making to each other's code or to the code in general. Challenges Encountered

GitHub appears to be a very helpful platform for programmers, but as complete beginners we encountered several challenges.

  • We found it daunting to merge our branches because we were afraid of potentially breaking the code or causing code overlaps.
  • There were many unfamiliar terms for us, such as merge, branch, commit, and repository, which made it difficult to understand what we were clicking on or the function of each feature.
  • We watched YouTube videos and asked teaching assistants for help, but found the pace too fast. Ultimately, the only way we managed to solve our issues was through trial and error until we started to comprehend the platform's functionalities.
